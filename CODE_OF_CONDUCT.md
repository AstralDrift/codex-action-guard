# Code of conduct

This project should be a useful, low-drama place to improve Codex GitHub Action safety.

## Expected behavior

- Be respectful and direct.
- Assume good intent while still being precise about risks.
- Keep technical disagreement focused on evidence.
- Make space for maintainers, security engineers, DevOps engineers, and first-time contributors.
- Report false positives and unsafe examples without shaming the people who wrote the workflow.

## Unacceptable behavior

- Harassment, insults, threats, or discriminatory language.
- Publishing private information or secrets.
- Repeated off-topic arguments after a maintainer asks to move on.
- Using the project to target maintainers or users of other projects.

## Enforcement

Maintainers may edit, hide, or delete comments; close issues; limit participation; or block users when needed to keep the project healthy.

If you need to report conduct concerns, contact a repository maintainer privately through GitHub.
